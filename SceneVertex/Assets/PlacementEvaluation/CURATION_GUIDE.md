# 큐레이션 재작성 가이드

이 문서는 `placement_elements.json` 은 유지한 채 `placement_layout_curated.json` 만 다시 큐레이트할 때 쓰는 전용 가이드다.

## 1. 이 문서의 목적

- 요소 정의는 충분하다고 가정한다.
- 문제는 "무엇을 그릴 수 있는가"가 아니라 "무슨 구조로 보이게 할 것인가"에 있다고 본다.
- 따라서 수정 우선순위는 항상 `layout` 이다.

## 2. 작업 시작 전 판단

먼저 아래 질문을 본다.

- 지금 어색한 이유가 색 문제인가
- 지금 어색한 이유가 크기 문제인가
- 아니면 구조와 흐름 문제인가

대부분의 경우 정답은 구조와 흐름 문제다. 이 경우 `placement_elements.json` 은 건드리지 않는다.

## 3. 현재 스키마에서 실제로 조정해야 하는 것

### `meadow`

- 화면에서 어디를 비울지 정한다.
- 레이아웃이 답답하면 `openCenterRadiusX`, `openCenterRadiusY` 를 키운다.
- 화면이 너무 심심하면 `accentGrassCount` 를 조금 올린다.

### `ponds`

- 큰 포컬 포인트를 만든다.
- 연못 실루엣이 단조로우면 `lobes` 를 조정한다.
- 물가 층이 약하면 `sandWidth`, `pebbleWidth` 를 조정한다.

### `trails`

- 장면 리듬을 만든다.
- 제일 먼저 손볼 곳이다.
- 길 수가 많다고 좋아지지 않는다.
- `points` 수보다 `흐름 관계`가 중요하다.

좋은 trail 기준:

- 메인 흐름 1개
- 보조 흐름 1~3개
- 약한 접근로 1~2개
- 갈라짐이 있더라도 다시 합쳐질 수 있어야 함
- 연못을 정면 관통하기보다 스치고 지나가야 함

### `groves`

- 큰 질량과 프레이밍을 만든다.
- 나무를 개별로 늘리기보다 군집의 위치와 크기를 먼저 조정한다.
- 나무가 너무 많으면 `treeCount` 보다 `radiusX`, `radiusY` 와 `center` 를 먼저 본다.

### `scatterZones`

- 보조 리듬을 만든다.
- 큰 구조를 해결하기 위한 수단이 아니라 마지막 보정용이다.
- 바위는 경계와 리듬 보정
- 덤불은 군집 연결
- 버섯은 작은 포인트

### `fenceLines`

- 가장자리 정리용이다.
- 경계를 암시하는 정도로만 쓴다.
- 중앙 구도를 해결하려고 펜스를 넣으면 대부분 실패한다.

## 4. 다시 큐레이트할 때 순서

1. `meadow` 로 비울 공간을 정한다.
2. `ponds` 로 메인 포컬 포인트를 잡는다.
3. `trails` 로 통행 흐름을 잡는다.
4. `groves` 로 좌우 질량을 맞춘다.
5. `scatterZones` 로 바위, 작은 덤불, 버섯을 보정한다.
6. `fenceLines` 로 가장자리만 정리한다.

## 5. 실패 패턴

- 길을 너무 많이 넣어서 도로망처럼 보이는 경우
- 길이 모두 비슷한 세기라서 우선순위가 없는 경우
- 연못과 길이 서로 무관하게 겹쳐 보이는 경우
- 숲 군집이 프레임이 아니라 점무늬처럼 흩어진 경우
- 바위와 버섯이 장면 구조를 해결하려고 과하게 쓰인 경우

## 6. 좋은 결과를 위한 실전 기준

- 멀리서 봤을 때 먼저 보이는 것은 `연못`, `개활지`, `통행 흐름` 이어야 한다.
- 중간 거리에서는 `숲 덩어리` 와 `길 어깨` 가 읽혀야 한다.
- 가까이서 봤을 때만 바위, 버섯, 자갈 디테일이 보여야 한다.
- 결과가 "설계된 길"처럼 보이면 실패다.
- 결과가 "계속 밟혀서 생긴 흔적"처럼 보이면 성공에 가깝다.

## 7. 요청별 대응 요약

### "좀 더 자연스럽게"

- `trails` 수를 줄이거나 `traffic` 차이를 키운다.
- `groves` 중심을 경계 쪽으로 더 밀어 프레임을 만든다.

### "목초지 길 느낌을 더"

- `trails[].soilExposure` 를 과하게 올리지 않는다.
- `braid` 와 `wander` 를 적절히 사용해 반복 통행 흔적처럼 보이게 한다.

### "연못이 더 좋아 보여야 해"

- 연못 크기보다 `lobes` 와 물가 층을 먼저 본다.
- 연못 주변 trail 이 너무 직선이면 같이 손본다.

### "오른쪽이 너무 무겁다"

- `groves` 와 `scatterZones` 를 본다.
- 펜스 개수를 줄여서 해결하려고 하지 말고, 먼저 큰 질량을 조정한다.

## 8. 최종 확인

- 지금 수정이 `layout` 문제였는가
- 그런데도 `elements` 를 건드리지는 않았는가
- 길, 연못, 군집, 경계의 역할이 서로 겹치지 않는가
- 이전보다 구조가 더 읽히는가
