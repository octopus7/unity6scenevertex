# 큐레이션 재작성 가이드

이 문서는 `placement_elements.json` 은 유지한 채 `placement_layout_curated.json` 만 다시 큐레이트할 때 사용하는 전용 가이드다.

## 1. 이 문서의 목적

- 요소 정의는 이미 충분하다고 가정한다.
- 수정 대상은 배치와 구도와 색 선택만이다.
- 즉, 기본 반경, 기본 세그먼트 길이, 배리에이션 수는 유지하고 장면 구성만 다시 만든다.

## 2. 작업 시작 전 판단

아래 질문에 모두 `예` 라면 `placement_elements.json` 은 건드리지 않는다.

- 요소 종류가 부족하지 않은가
- 각 요소가 3가지 색 배리에이션을 이미 갖고 있는가
- 크기 자체가 틀린 것이 아니라 배치가 어색한가
- 문제의 본질이 구도, 밀도, 동선, 색 조합에 가까운가

하나라도 `아니오` 라면 `JSON_EDIT_RULES.md` 를 다시 보고 요소 정의 수정 여부를 판단한다.

## 3. 재큐레이션 기본 원칙

- 먼저 장면의 포컬 포인트를 정한다.
- 다음으로 동선을 정한다.
- 그다음 큰 질량인 나무를 배치한다.
- 이후 바위와 덤불로 중간 밀도를 조정한다.
- 마지막에 버섯과 팬스로 디테일을 잡는다.

## 4. 현재 샘플 장면의 기준 구조

- 좌측 물웅덩이가 주 포컬 포인트다.
- 좌하단에서 우중앙으로 이어지는 길이 시선 흐름이다.
- 좌상단과 우측 외곽의 나무 군집이 프레임이다.
- 물가의 바위가 포컬 포인트 외곽을 고정한다.
- 우측 팬스는 화면 경계를 정리하는 보조 장치다.

새 큐레이션을 만들 때도 이 구조를 완전히 버리기보다 비슷한 역할 관계를 유지하는 편이 안전하다.

## 5. 배열별 수정 원칙

### 5-1. `typeVariants`

- 장면 전체 색감 수정용이다.
- 색 분위기만 바꾸고 싶을 때는 여기부터 본다.
- 같은 요소 안에서 일부만 다른 색을 쓰는 방식은 금지한다.

### 5-2. `groundPatches`

- 가장 먼저 포컬 포인트와 바닥 층을 다시 잡는다.
- 물을 바꾸면 모래와 자갈도 같이 바꾼다.
- 흙은 길과 나무 밑의 압축감을 보조하는 용도로만 쓴다.
- 잔디 패치는 전체 배경을 살짝 흔들어 주는 용도다.

### 5-3. `roadSegments`

- 세그먼트 하나씩 보지 말고 전체 흐름으로 본다.
- 인접 세그먼트끼리 각도 차가 급하면 인공적으로 보인다.
- 길은 너무 중앙을 오래 점유하지 않게 한다.

### 5-4. `props`

- 나무는 프레임과 질량 담당이다.
- 바위는 경계와 시선 멈춤점 담당이다.
- 덤불은 큰 요소와 작은 요소 사이 완충층이다.
- 버섯은 가장 마지막에 조절한다.

### 5-5. `fenceSegments`

- 화면의 오른쪽이나 위쪽 외곽을 정리하는 용도만 허용한다.
- 장면 중심부로 들어오지 않게 한다.
- 팬스 개수가 늘어나면 시선이 분산되므로 매우 보수적으로 조정한다.

## 6. 재큐레이션 절차

1. `typeVariants` 를 보고 현재 색감이 목표와 맞는지 먼저 판단한다.
2. `groundPatches` 에서 물과 주변 경계를 다시 잡는다.
3. `roadSegments` 로 길 흐름을 재조정한다.
4. `props` 의 나무를 먼저 옮기거나 줄인다.
5. 바위와 덤불로 균형을 보정한다.
6. 버섯과 팬스를 맨 마지막에 다듬는다.
7. 빈 공간이 남아 있는지 확인한다.

## 7. 좋은 재큐레이션의 판단 기준

- 1초 안에 포컬 포인트가 읽힌다.
- 길이 자연스럽게 시선을 움직인다.
- 큰 오브젝트가 서로 경쟁하지 않는다.
- 색이 튀는 요소가 너무 많지 않다.
- 좌우 무게 균형은 맞지만 완전 대칭은 아니다.
- 물, 모래, 자갈, 흙, 잔디가 층으로 읽힌다.
- 버섯은 가까이 보면 보이지만 멀리서는 과하게 튀지 않는다.

## 8. 재큐레이션 시 흔한 실패 패턴

- 모든 요소를 고르게 퍼뜨리는 것
- 길을 지나치게 직선으로 만드는 것
- 나무를 중앙에 몰아 답답하게 만드는 것
- 팬스를 너무 많이 써서 배경이 아니라 피사체가 되게 만드는 것
- 물과 모래와 자갈이 따로 노는 것
- 덤불과 버섯의 관계를 끊어 작은 디테일이 붕 뜨는 것

## 9. 요청 유형별 대응

### "더 자연스럽게"

- `props` 의 규칙적인 간격부터 깨고, 덤불과 버섯을 나무 주변으로 다시 묶는다.
- `roadSegments` 의 회전 변화가 부드러운지 점검한다.

### "더 예쁘게"

- `typeVariants` 에서 전체 색감을 먼저 다듬는다.
- 포컬 포인트 주변 대비를 높이고 나머지 영역의 경쟁 요소를 줄인다.

### "너무 복잡해"

- 나무 1~2개 또는 바위 1~2개를 제거한다.
- 팬스 구간을 짧게 한다.
- 중앙과 길 주변의 덤불 밀도를 낮춘다.

### "왼쪽이 심심해"

- 물 주변 `sand` 와 `pebble` 배치를 풍부하게 한다.
- 물가 바위와 작은 덤불 포인트를 추가한다.

### "오른쪽이 무거워"

- 우측 나무 개수와 팬스 길이를 먼저 줄인다.
- 우측 상단 큰 질량 하나를 외곽으로 빼거나 제거한다.

## 10. 재큐레이션 결과 전달 형식

나중에 사용자나 에이전트가 장면을 다시 요청할 때는 아래 형식으로 생각하면 된다.

- 유지할 것
  - 요소 정의
  - 해상도
  - 스키마
- 바꿀 것
  - `typeVariants`
  - `groundPatches`
  - `roadSegments`
  - `props`
  - `fenceSegments`
- 이유
  - 포컬 포인트
  - 흐름
  - 밀도
  - 색 조화

## 11. 최종 확인

- 요소 정의를 바꾸지 않고 원하는 개선이 되었는가
- 단순히 개수만 늘리거나 줄인 것이 아니라 관계가 좋아졌는가
- 길과 포컬 포인트와 프레임이 함께 읽히는가
- 기존보다 더 설계된 장면처럼 보이는가
